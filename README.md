# oni-antigravity-prompts

**Система промптов для Antigravity, которая не превращается в кашу.**

Заменяет "эволюционно наросший" GEMINI.md-монстр на три уровня:
- **Глобальные правила** — кто я, чего хочу, какой стек
- **Skills** — переиспользуемые "программы" для типовых задач (brainstorm / debug / ship)
- **Сессионные брифы** — что делаем конкретно сегодня

---

## Зачем это

Если у тебя:
- 🐙 **GEMINI.md превратился в простыню** на 500 строк, агент половину игнорирует
- 🧩 **Правила размазаны** по 10 файлам, непонятно где что править
- 🔁 **Каждый раз** заново формулируешь "сделай фичу", "почини баг" — и каждый раз по-разному
- 🏷️ **Промпты привязаны к одной IDE** — перешёл с Cursor → потерял всё
- 🤷 **Агент делает не то** что ты имеешь в виду

→ Эта система чинит все 5 болей.

## Три уровня иерархии

```
┌─────────────────────────────────────────────────────────┐
│ L1 — ГЛОБАЛЬНЫЙ  (всегда загружен Antigravity)         │
│                                                          │
│  .agent/                                                 │
│  ├── IDENTITY.md      ← кто я, мой уровень, мой стиль   │
│  ├── PRINCIPLES.md    ← неизменные правила (кратко!)    │
│  ├── STACK.md         ← технологии и сервисы             │
│  └── STYLE.md         ← как общаемся, язык, формат       │
└─────────────────────────────────────────────────────────┘
                          ↓ ссылается на
┌─────────────────────────────────────────────────────────┐
│ L2 — ПРОЕКТНЫЙ  (для конкретного проекта)               │
│                                                          │
│  project/                                                │
│  ├── PROJECT.md       ← что за проект, зачем, для кого  │
│  ├── ARCHITECTURE.md  ← архитектура этого проекта       │
│  └── CONVENTIONS.md   ← code style, naming, паттерны    │
└─────────────────────────────────────────────────────────┘
                          ↓ активирует на сессию
┌─────────────────────────────────────────────────────────┐
│ L3 — СЕССИОННЫЙ  (что делаем сегодня)                   │
│                                                          │
│  session/SESSION_BRIEF.md                                │
│  ├── Цель сессии (1-3 строки)                            │
│  ├── Конкретные задачи (bullet list)                      │
│  └── Ограничения / known issues                          │
└─────────────────────────────────────────────────────────┘
                          ↓ вызывает
┌─────────────────────────────────────────────────────────┐
│ SKILLS  (каталог переиспользуемых процедур)             │
│                                                          │
│  .agent/skills/                                          │
│  ├── brainstorm.md    ← "давай сделаем X"               │
│  ├── feature.md       ← "сделай фичу"                   │
│  ├── debug.md         ← "X не работает"                 │
│  ├── review.md        ← "проверь мой код"               │
│  ├── refactor.md      ← "почисти код"                   │
│  └── ship.md          ← "задеплой"                      │
└─────────────────────────────────────────────────────────┘
```

## Быстрый старт (5 минут)

### 1. Скопируй в свой проект

```bash
# Из корня твоего проекта
cp -r ~/oni-antigravity-prompts/.agent .
cp -r ~/oni-antigravity-prompts/scripts .
mkdir -p project session
cp ~/oni-antigravity-prompts/project/* project/  # и отредактируй
```

### 2. Отредактируй L1 под себя

Открой `.agent/IDENTITY.md` — замени плейсхолдеры `[ИМЯ]`, `[УРОВЕНЬ]` на свои.

### 3. Заполни L2 под проект

`project/PROJECT.md` — что за проект, зачем, для кого, какие метрики успеха.

### 4. Начни сессию

```bash
./scripts/new-session.sh "Добавить OAuth Google"
# Создаст session/2026-06-30-oauth-google.md
```

Открой Antigravity, скажи:
> *"Прочитай `.agent/` и `session/2026-06-30-oauth-google.md`. Используй skill `feature`."*

### 5. Закрой сессию

```bash
./scripts/land-the-plane.sh
# Показывает что осталось в SESSION_BRIEF, предлагает сохранить
```

## Главные принципы

### 1. **L1 — короткий и неизменный**
GEMINI.md на 500 строк = невозможно поддерживать. Каждый файл L1 = 30-80 строк. Хочешь добавить правило — спроси себя "это правило для всех проектов или только для этого?". Если для всех — в L1. Если только этого — в L2.

### 2. **Skills — переиспользуемые**
Каждый skill = процедура, которая вызывается триггером ("сделай", "почини", "проверь"). Не пиши новый промпт "как сделать фичу" — вызови skill `feature`. Не дублируй.

### 3. **L3 — конкретный и временный**
SESSION_BRIEF живёт одну сессию. В конце сессии — переноси важное в L1/L2 (если про это правило должно жить дальше) или в Beads (если это таск на потом). Не копи сессионный мусор в постоянные правила.

### 4. **Vendor-neutral**
Эта система работает в Antigravity, Claude Code, Cursor. Разница только в том КАК IDE читает эти файлы:
- Antigravity: `.agent/` (наша конвенция)
- Claude Code: `CLAUDE.md` или `.claude/`
- Cursor: `.cursorrules`

Содержимое — одно и то же. Только конвенция папки меняется.

### 5. **Skills enforced, не negotiated**
Текстовые правила агент может игнорировать. Skills — это структурированные процедуры с обязательными шагами. Агент **должен** следовать. Это разница между "пожалуйста, пиши тесты" (часто игнорирует) и "skill `feature` требует шаг 5: написать тесты" (нельзя пропустить).

---

## Когда НЕ использовать эту систему

- **Pet-проект на 1 вечер** — overkill, пиши GEMINI.md в 30 строк
- **Ты senior-разработчик и сам делаешь review** — тебе Skills не нужны, ты и так помнишь workflow
- **Один проект, никогда не вырастет** — L2 достаточно, L3 излишен

Эта система для **соло-не-программиста с растущим количеством проектов и агентских сессий**.
