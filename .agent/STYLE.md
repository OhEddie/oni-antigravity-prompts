# STYLE.md

**Как мы общаемся с агентом. Тон, язык, формат.**

---

## Язык

- **Основной:** русский
- **Код / коммиты / комментарии / названия переменных:** английский
- **Документация:** русский (для меня), английский (для всего что пойдёт в open source)
- **ADR / Beads:** английский (стандарт индустрии, agent-friendly)

## Тон

- **Кратко.** Без воды. Каждое предложение — информация, не filler.
- **Прямо.** Если плохо — "это плохо потому что X". Без "ну как бы", "возможно стоит подумать".
- **Конструктивно.** Вместе с критикой — вариант решения.

## Формат ответов

### Короткий вопрос → короткий ответ
> Я: "Какая версия Next.js в этом проекте?"
> Агент: "14.2.28"

### Сложный вопрос → структурированный ответ
> Я: "Как работает auth в этом проекте?"
> Агент:
> - **Provider:** NextAuth.js credentials + JWT
> - **DB:** users table в Prisma
> - **Session:** JWT в httpOnly cookie
> - **API:** Bearer token через `Authorization` header
> - **См. также:** `lib/auth.ts:42`

### Когда сделал работу → итог
> Я: "Сделай bd-007"
> Агент (в конце):
> ✅ **Сделано:** добавил endpoint `/api/v1/users/:id/avatar`, тесты прошли, build OK
> 📋 **Изменено:** `app/api/v1/users/[id]/avatar/route.ts:1-58`, `__tests__/avatar.test.ts:1-32`
> 🐛 **Нашёл:** баг в `lib/upload.ts:14` — не проверяется MIME type. Создал bead bd-008.
> ⚠️ **Требует решения:** хранить ли аватарки в Supabase S3 или внешнем CDN. Создал ADR-0003 в Proposed.

## Чего НЕ делать

- ❌ Не используй emoji в коде, коммитах, именах файлов
- ❌ Не пиши "как было сказано выше" / "следует отметить" — filler words
- ❌ Не выдумывай API/функции которых нет
- ❌ Не говори "я не знаю, но попробую угадать" — скажи "не знаю, проверю"
- ❌ Не давай 10-страничный ответ на 1-строчный вопрос

## Формат кода

- **Naming:** camelCase (TS/JS), snake_case (Python), kebab-case (файлы и URL)
- **Imports:** сортированы (ESLint import/order)
- **Функции:** ≤ 50 строк, иначе разбить
- **Файлы:** ≤ 300 строк, иначе разбить
- **Comments:** объясняют **ЗАЧЕМ**, не **ЧТО**. Код говорит что, коммент — зачем.

## Формат commit messages

**Conventional commits:** `<type>(<scope>): <description>` + body + footer.

- **Types:** `feat`, `fix`, `refactor`, `docs`, `test`, `chore`, `perf`, `ci`
- **Description:** ≤ 72 символов, lowercase, без точки
- **Body:** объясни ЗАЧЕМ
- **Footer:** `Refs: bd-NNN`, `Closes: #N`, `BREAKING CHANGE: ...`

```
feat(auth): add OAuth Google provider

Users жаловались на пароли. OAuth быстрее.

Refs: bd-014
```
