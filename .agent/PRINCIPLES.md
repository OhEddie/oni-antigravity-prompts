# PRINCIPLES.md

**Неизменные правила. Только то что верно для ВСЕХ проектов навсегда.**

Если сомневаешься "это в PRINCIPLES или в project/CONVENTIONS?" — спроси:
- "Это правило нарушит работу если его убрать в другом проекте?" — Нет → в CONVENTIONS
- "Это правило должно жить через год независимо от проекта?" — Да → в PRINCIPLES

---

## 1. Код

- **Тесты обязательны** для любой логики > 20 строк. Не "потом напишу" — сразу.
- **Никакого `console.log()` и `print()` в продакшн-коде.** Используй `logger.debug()` / `logger.info()`.
- **Никаких TODO без bead.** Увидел TODO — создай bead, укажи в нём где TODO, удали TODO из кода.
- **Никаких коммитов с секретами.** `.env` в `.gitignore`, секреты — в Secret Manager / Vault.
- **Любая миграция БД = отдельный PR** с явным `db:migrate` шагом.

## 2. Архитектура

- **Не меняй публичный API контракт** без major version bump и ADR.
- **Новые зависимости = обсуждение.** Не добавляй 5 пакетов чтобы сделать одну вещь.
- **Архитектурное решение = ADR.** Создай `docs/adr/NNNN-title.md`, статус Proposed → Accepted.
- **Микросервисы — нет** для проектов < 10 разработчиков. Монолит правильнее.

## 3. Коммуникация

- **Говори что сделал, что осталось, что нашёл.** Не "сделано" — а "сделано: X, осталось: Y, нашёл баг: Z (создал bead bd-007)".
- **Если сомневаешься — скажи прямо.** "Я бы выбрал X, но есть риск Y, обсудим?"
- **Если не знаешь — скажи "не знаю, надо проверить".** Не выдумывай API/функции.

## 4. Workflow

- **Каждая сессия начинается с чтения** `.agent/`, `project/PROJECT.md`, и `session/SESSION_BRIEF.md` (если есть).
- **Каждая сессия заканчивается `bd sync`** (если используется Beads) и `git push`.
- **Каждое нетривиальное изменение = branch + PR.** Не коммить в main напрямую.
- **Каждый PR проходит через skill `review`** перед merge.

## 5. Безопасность

- **Принцип минимальных привилегий.** API keys с минимально нужными scopes.
- **Никаких SQL-запросов через конкатенацию строк.** Только parameterized queries.
- **Никаких user input в shell-командах.** Используй `subprocess.run([...])` с массивом, не `shell=True`.
- **Env vars с чувствительными данными НИКОГДА не коммитятся.** Случайно закоммитил — ротируй ключ немедленно.

## 6. Стоимость

- **Прежде чем добавить платный сервис — посчитай monthly cost** при 10x и 100x масштабе.
- **Free tier first.** Если есть разумный free tier — начинай с него.
- **Не плати за то что можно сделать скриптом** за 1 час.

**Эти 6 категорий = 30 правил суммарно. Не больше.** Если список растёт — значит часть правил не про "принципы", а про проект. Переноси в `project/CONVENTIONS.md`.
